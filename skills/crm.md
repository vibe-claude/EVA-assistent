# CRM и базы данных

Этот скилл читается когда пользователь просит создать базу клиентов, задач, записей или любую другую структурированную базу данных.

---

## Создание базы данных

**Триггеры:** "создай базу клиентов", "создай базу задач", "хочу вести учёт", "сделай мне таблицу", "база данных", "нужна CRM"

### Шаг 1 — Уточнить структуру

Не создавай базу сразу. Сначала предложи структуру и согласуй:

> "Понял. Предлагаю такую структуру для базы клиентов:
> — Имя
> — Телефон
> — Дата последнего визита
> — Следующий визит
> — Заметки
>
> Что добавить, убрать или изменить?"

Жди ответа. Учти правки. Только после подтверждения — создавай.

### Шаг 2 — Создать SQLite базу

```bash
mkdir -p ~/data
sqlite3 ~/data/clients.db << 'SQL'
CREATE TABLE IF NOT EXISTS clients (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  name TEXT NOT NULL,
  phone TEXT,
  last_visit DATE,
  next_visit DATE,
  notes TEXT,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
SQL
echo "База создана: ~/data/clients.db"
```

Имя файла и таблицы — по смыслу (clients.db, tasks.db, orders.db и т.д.).

### Шаг 3 — Показать что создано

После создания: показать структуру и объяснить как добавлять записи:

> "База создана. Чтобы добавить клиента — напиши мне: 'добавь клиента Имя, телефон, дата визита'"

---

## Добавление записи

**Триггеры:** "добавь клиента", "запиши клиента", "новый клиент"

```bash
sqlite3 ~/data/clients.db \
  "INSERT INTO clients (name, phone, last_visit, notes) VALUES ('ИМЯ', 'ТЕЛЕФОН', 'ДАТА', 'ЗАМЕТКИ');"
echo "✅ Добавлено"
```

---

## Просмотр базы

**Триггеры:** "покажи клиентов", "список клиентов", "кто записан"

```bash
sqlite3 -column -header ~/data/clients.db \
  "SELECT name, phone, next_visit, notes FROM clients ORDER BY next_visit;"
```

---

## Напоминания о визитах

Если в базе есть поле `next_visit` — джоб может проверять кому пора написать:

```bash
TODAY=$(date +%Y-%m-%d)
TOMORROW=$(date -d "+1 day" +%Y-%m-%d)
sqlite3 ~/data/clients.db \
  "SELECT name, phone, next_visit FROM clients WHERE next_visit = '$TOMORROW';"
```

Если есть клиенты — написать пользователю: "Завтра визит у: [список]"

---

## Правила

- База хранится в `~/data/` (создаётся автоматически)
- Структуру всегда согласовывать до создания
- После создания базы — записать в `wiki/log.md` что и когда создано
- Если пользователь хочет изменить структуру позже — использовать `ALTER TABLE`
