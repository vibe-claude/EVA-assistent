## Обновить статус документа в wiki

Триггеры:
- "получили ТУ по <объекту>"
- "договор по <объекту> закрыт / подписан / готов"
- "акт подписан по <объекту>"
- "согласовали <объект>"
- "отказали по <объекту>"
- "закрой заявку по <объекту>"
- любая фраза об изменении статуса по объекту/документу

Алгоритм:
1. Найти файл(ы) в wiki по объекту и типу:
```bash
grep -rl "<объект>" /home/claudeagent/eva/wiki/ --include="*.md" | xargs grep -l "status: в работе"
```
2. Определить новый статус из сообщения: получено / выполнено / отказано
3. Обновить frontmatter:
```bash
sed -i "s/^status: в работе$/status: <новый статус>/" <файл>
```
4. Добавить запись в раздел `## История`:
```
- **YYYY-MM-DD** — <суть изменения по словам Александра>
```
5. Обновить статус в index.md
6. Подтвердить: "Статус обновлён: <название> → <новый статус>"

## Найти информацию по объекту или сотруднику
Триггеры: "найди всё по <объект/человек>", "информация по <объект>", "что есть по <объект>", "покажи всё по <объект>", "справка по <объект>", "задачи <сотрудника>"

Алгоритм — проверить ВСЕ ТРИ источника:

**Шаг 1 — Notion (задачи):**
```bash
source /home/claudeagent/eva/.env
curl -s -X POST "https://api.notion.com/v1/databases/58eec372c06483a58a51819f2b6584c4/query" \
  -H "Authorization: Bearer $NOTION_TOKEN" \
  -H "Notion-Version: 2022-06-28" \
  -H "Content-Type: application/json" \
  -d '{
    "filter": {
      "and": [
        {"property": "Задачи", "title": {"contains": "<название объекта>"}},
        {
          "or": [
            {"property": "Status", "status": {"equals": "В работе"}},
            {"property": "Status", "status": {"equals": "Контроль!!!"}},
            {"property": "Status", "status": {"equals": "В плане"}}
          ]
        }
      ]
    }
  }'
```
Показывать только статусы: В работе, Контроль!!!, В плане. **НЕ показывать: Выполнено, Архив.**

**Шаг 2 — Wiki (письма, документы, история):**
```bash
grep -rl "<slug объекта>" /home/claudeagent/eva/wiki/ --include="*.md"
```
Прочитай найденные страницы и покажи историю событий.

**Шаг 3 — Google Sheets (задачи сотрудников):**
```bash
curl -sL "https://docs.google.com/spreadsheets/d/1ACd6DhQFF1z9FswqtlqQswEleiLRtJ2nTYeSAQVorz8/export?format=csv&gid=151402854" | python3 -c "
import csv, sys, io
query = '<название объекта или сотрудника>'.lower()
data = sys.stdin.read()
reader = csv.DictReader(io.StringIO(data))
for row in reader:
    obj = (row.get('Наименование объекта') or '').lower()
    responsible = (row.get('Ответственный') or '').lower()
    if query in obj or query in responsible:
        print(f\"{row.get('Наименование объекта')} | {row.get('Задача')} | {row.get('Статус задачи')} | {row.get('Ответственный')}\")
"
```

**Формат ответа:**
```

• Задача — Ответственный (срок)

📂 Документы и история (wiki):

📊 Задачи в таблице отдела (Google Sheets):
```

Блоки без данных — пропустить. **Никогда не говорить "нет данных" не проверив все три источника.**
